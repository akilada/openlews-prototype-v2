# Contributing to OpenLEWS

First off, thank you for considering contributing to this project! It's people like you that make the open-source community such an amazing place to learn, inspire, and create.

## 1. Getting Started

### Reporting Bugs
If you find a bug, please create a new issue. Be sure to include:
* Steps to reproduce the bug.
* Expected behavior vs. actual behavior.
* Screenshots or logs if applicable.

### Suggesting Enhancements
Open an issue to discuss your idea before writing code! This saves time and ensures your idea aligns with the project goals.

## 2. Development Workflow

We follow a strict **Feature Branch Workflow**. Please do not commit directly to the `main` branch.

1.  **Fork** the repository.
2.  **Clone** your fork locally.
3.  **Create a Branch** for your specific change:
    * For features: `git checkout -b feature/my-new-feature`
    * For bug fixes: `git checkout -b fix/issue-number-bug-name`
4.  **Make your changes** and commit them.
5.  **Push** to your fork.

## 3. Pull Request Process

When you are ready to submit your changes:

1.  Open a **Pull Request** against the `main` branch of this repository.
2.  Fill out the PR description clearly.
3.  **Code Review:** The repository owner must review and approve your changes before they can be merged.
    * *Note: PRs without approval cannot be merged.*
4.  Once approved, the owner will merge your changes.

## 4. Coding Standards
* Keep code clean and commented where necessary.
